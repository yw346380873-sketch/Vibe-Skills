# VCO Release v2.3.45

- Date: 2026-03-15
- Commit(base): 78856fb

## Highlights

- Hardened the `benchmark_autonomous` runtime bridge for Windows-hosted automation. The runtime now verifies that critical stage artifacts are durable before returning, rather than assuming every consumer sees freshly written files immediately.
- Added artifact-root-relative path fields to `runtime-summary.json`. This makes bridge consumers resilient when `powershell.exe` stdout carries lossy absolute-path encoding on systems with non-ASCII user profile paths.
- Preserved the real bounded XL executor introduced in `v2.3.44`: benchmark mode still runs repo-safe execution units, writes per-unit receipts and logs, emits `execution-manifest.json`, and produces a green `benchmark-proof/manifest.json`.
- Restored the Python runtime bridge proof so temporary artifact-root execution is verified end to end instead of only through PowerShell-native gates.

## Migration Notes

- Consumers that invoke `invoke-vibe-runtime.ps1` from Python or other non-PowerShell hosts should prefer the new relative summary paths when reconstructing files under a known `ArtifactRoot`.
- No public runtime stages changed. The six-stage governed runtime contract remains intact; this release only hardens the bridge and proof surfaces.

# 理论框架构建指南

## 组合类型识别

### 并联型 (Parallel)
A和B处理不同方面然后融合。

统一形式:
```
f(x) = α·g_A(x) + (1-α)·g_B(x), α ∈ [0,1]
```
- 当α=1时退化为A
- 当α=0时退化为B
- 扩展: 使用可学习的α（注意力机制）或Dirichlet分布

适用场景: 两种方法处理同一输入但关注不同特征

### 串联型 (Serial)
A的输出是B的输入。

统一形式:
```
f(x) = g_B(g_A(x))
```
框架: 定义通用变换管道 T = T_n ∘ ... ∘ T_2 ∘ T_1
- A和B是管道中的特定变换
- 可以在管道中插入新的变换C

适用场景: 一种方法的输出自然成为另一种方法的输入

### 嵌套型 (Nested)
B嵌入A的某个模块中。

统一形式:
```
f(x) = g_A(x; module=g_B)
```
框架: 定义可插拔的模块化架构
- A是主框架，B是可替换模块
- 可以定义模块接口，允许不同的B实现

适用场景: 一种方法的某个组件可以被另一种方法替换

## α分析

### α的物理/数学含义
- 正则化参数: 控制两种方法的相对贡献
- 偏差-方差权衡: α调节模型复杂度
- 信息融合权重: 控制两个信息源的融合比例

### 为什么α=0.5附近效果更好的理论解释

1. **信息论角度**: 最大化两个分布的互信息
   - 当α=0.5时，组合分布的熵最大
   - 信息利用最充分

2. **正则化角度**: 最优偏差-方差权衡
   - α=1或α=0时，模型偏向单一方法（高偏差或高方差）
   - α=0.5时达到最优权衡点

3. **博弈论角度**: 纳什均衡
   - 将A和B视为两个玩家
   - 混合策略的纳什均衡通常在中间值

4. **集成学习角度**: 多样性最大化
   - 当两个模型权重相等时，集成的多样性最大
   - 错误的相关性最低

## Non-trivial论证模板

### 模板1: 理论Non-trivial
```
组合不是简单加法，存在交互项:
f(x) = α·A(x) + (1-α)·B(x) + α(1-α)·h(A(x), B(x))
其中h是交互函数，捕捉A和B之间的协同效应。
当α∈{0,1}时交互项消失，只有α∈(0,1)时才能获得协同增益。
```

### 模板2: 实验Non-trivial
```
性能曲线在α∈(0,1)时超过线性插值:
Performance(α) > α·Performance(A) + (1-α)·Performance(B)
这说明组合产生了超加性效果（superadditive），
不能简单地通过加权平均来解释。
```

### 模板3: 问题Non-trivial
```
A+B解决了A和B各自无法解决的问题:
- 问题P需要同时满足条件C1和C2
- A满足C1但不满足C2
- B满足C2但不满足C1
- A+B同时满足C1和C2
```

### 模板4: 计算Non-trivial
```
组合后需要新的优化算法:
- A的优化目标: min L_A(θ_A)
- B的优化目标: min L_B(θ_B)
- 组合优化: min α·L_A(θ) + (1-α)·L_B(θ) + λ·R(θ)
其中R(θ)是正则化项，确保θ同时适配A和B。
这个优化问题不能分解为独立优化A和B。
```

## A+B+C扩展策略

### 从矩阵中寻找C
- 在Phase 2的评分矩阵中，找到与A和B都有高互补性的论文C
- 条件: score(A,C) >= 3.0 AND score(B,C) >= 3.0

### 扩展框架
```
f(x) = α·A(x) + β·B(x) + (1-α-β)·C(x)
约束: α + β + (1-α-β) = 1, α,β ≥ 0, α+β ≤ 1
```
这是单纯形(simplex)约束，参数空间是一个三角形。

### 高级扩展: 注意力权重
```
weights = softmax([w_A, w_B, w_C])
f(x) = weights[0]·A(x) + weights[1]·B(x) + weights[2]·C(x)
```
权重可以是:
- 固定的（超参数搜索）
- 可学习的（端到端训练）
- 输入依赖的（注意力机制）

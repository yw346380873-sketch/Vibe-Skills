# 发布 v2.3.50 执行计划

**日期**: 2026-03-26
**需求文档**: [2026-03-26-release-v2.3.50.md](../requirements/2026-03-26-release-v2.3.50.md)

## 设计

1. 以 `v2.3.49` 为上一版本基线
2. 汇总 `main` 自上一 tag 以来的主要变化，生成 `v2.3.50` release notes
3. 更新受 `config/version-governance.json` 约束的版本面
4. 通过分支 + PR 收敛到 `main`
5. 在合并后的 `main` 上切 tag 并创建 GitHub release

## 验证

1. 版本标记一致
2. `git diff --check` 通过
3. 发布说明与实际 main 变更相符
4. phase 结束后不遗留临时文件

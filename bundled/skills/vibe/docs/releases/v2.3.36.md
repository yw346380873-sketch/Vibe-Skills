# VCO Release v2.3.36

- Date: 2026-03-13
- Commit(base): 9a093e2

## Highlights

- Hardened the install-facing story so "full-featured" now means governed closure of the repository-owned surface, not an overclaim that every host plugin, MCP surface, and secret is auto-provisioned.
- Added explicit host-plugin policy documents for Chinese and English operators:
  - `docs/install/host-plugin-policy.md`
  - `docs/install/host-plugin-policy.en.md`
- Added reusable full-featured install prompt bundles for Windows and Linux operators:
  - `docs/install/full-featured-install-prompts.md`
  - `docs/install/full-featured-install-prompts.en.md`
- Updated the README, English README, cold-start onboarding, and one-shot setup docs so first-time users can choose among `minimum viable`, `recommended full-featured`, and `enterprise-governed` paths without guessing.
- Recut the governed release surface to `2.3.36` and aligned release-facing metadata across governance source, manifests, changelog, and runtime-neutral freshness tests.

## Migration Notes

- No runtime contract was widened in this patch release. Official runtime ownership, host plugin ownership, and provider secret ownership remain unchanged.
- Linux and macOS operators still need `pwsh` to obtain the authoritative freshness / coherence gate path. Without `pwsh`, installation remains supported but degraded, and some gates are warning-only.
- At the time of this release on 2026-03-13, Codex remained `supported-with-constraints`, Claude Code remained `preview`, and OpenCode was still `not-yet-proven`.
- Later work on 2026-03-26 promoted OpenCode to a truthful `preview` adapter lane with dedicated install/check docs and proof notes. Read the current status from `docs/install/opencode-path*.md` and `docs/universalization/host-capability-matrix.md`, not from this historical release note alone.

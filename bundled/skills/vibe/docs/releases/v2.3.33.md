# VCO Release v2.3.33

- Date: 2026-03-13
- Commit(base): 370ec76

## Highlights

- Fixed one-shot bootstrap provider seeding so existing `settings.json` values for `OPENAI_API_KEY` and `ARK_API_KEY` are detected and reused instead of producing misleading missing-key warnings.
- Added clearer operator messaging for the external CLI bootstrap path: `claude-flow` installation via `npm` can legitimately take several minutes, and deprecated warnings are advisory unless the command exits non-zero.
- Aligned deep runtime checking with mirror governance so optional `nested_bundled` surfaces no longer generate false bundled-config warnings during `check.ps1 -Deep` / one-shot doctor execution.
- Resynced canonical and bundled mirrors and regenerated `config/skills-lock.json` to keep release packaging parity intact after the bootstrap fixes.

## Migration Notes

- No user migration is required.
- Existing installs can be refreshed by re-running the governed bootstrap or by syncing the latest `skills/vibe` payload into the target Codex root.
- Linux / macOS operators still get the best verification coverage when `pwsh` is installed; without it, shell bootstrap remains supported but authoritative PowerShell gates degrade to warnings by design.

# VCO Release v2.3.44

- Date: 2026-03-15
- Commit(base): 78856fb

## Highlights

- Upgraded `benchmark_autonomous` from a receipt-only `plan_execute` stage into a bounded local XL executor. The execute stage now runs real repo-safe benchmark units, captures stdout/stderr, records exit codes, and emits execution manifests instead of stopping at an `execution-contract-prepared` receipt.
- Added `config/benchmark-execution-policy.json` as the machine-readable benchmark scheduler contract. The benchmark runtime is now governed by explicit wave/unit topology, timeout policy, and proof thresholds.
- Added benchmark proof outputs under the runtime session root, including `execution-manifest.json`, per-unit execution results, and `benchmark-proof/manifest.json`.
- Strengthened runtime-neutral bridge tests and governed runtime contract assertions so benchmark mode must prove actual execution closure, not just requirement/plan/cleanup artifact existence.
- Added `scripts/verify/vibe-benchmark-autonomous-proof-gate.ps1` to verify benchmark proof closure directly from a fresh temporary runtime artifact root.

## Migration Notes

- `benchmark_autonomous` now means real governed execution closure. It still remains repo-safe and bounded, but it no longer stops at plan receipts alone.
- The benchmark runtime is intentionally configured to avoid recursive self-verification loops. The default profile executes repo-safe local commands that prove runtime closure without reinvoking the governed contract gate from inside the benchmark workload itself.
- This release does not change the six-stage public runtime model. It hardens stage 5 so the existing model becomes materially true for benchmark execution.
